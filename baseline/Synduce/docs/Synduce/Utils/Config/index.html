val node_failure_behavior : bool Base.ref

When `node_failure_behaviour` is set to `true` (the default value), the failure to solve a given configuration is considered as an unrealizability result. That is, the configuration will not be refined further. If `node_failure_behaviour` is set to `false`, the failure will only interpreted as realizable: subconfigurations will be explored.
